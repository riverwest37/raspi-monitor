## 概要

`raspi-monitor` は、Raspberry Pi を家庭内監視表示用の端末として使うためのサンプルです。

本リポジトリでは、テレビまたはモニターに接続した Raspberry Pi 上で Chromium を全画面表示し、画面を4分割して家庭内の情報を一覧表示します。

KDP本  
**「その気になれば Raspberry Piで作る家庭IoT実践入門」**  
の第7章「TV表示システム構築」と対応しています。

第6章では、Node-RED による MQTT イベント受信と CEC 制御の流れを説明しています。  
本リポジトリでは、その先の「テレビ画面に何をどのように表示するか」を扱います。

---

## 目的

家庭内のテレビを大型モニターとして活用し、次のような情報を1画面にまとめて表示することを目的としています。

```text
左上：車庫映像
右上：玄関映像
左下：環境情報・CO2・状態表示など
右下：スケジュール・通知・予定表示

これにより、玄関や車庫の映像、家庭内の状態、予定情報などを一目で確認できます。

ファイル構成
raspi-monitor/
├─ README.md
├─ quad.html
├─ schedule.html
├─ start_quad.sh
└─ update_schedule_api.py
ファイル説明
quad.html

画面を4分割して表示するHTMLファイルです。

Raspberry Pi の Chromium ブラウザでこのファイルを開くことで、テレビ画面を2行×2列に分割して表示します。

表示内容の例です。

左上：車庫カメラ映像
右上：玄関カメラ映像
左下：CO2濃度・環境情報
右下：スケジュール表示

車庫映像や玄関映像は、各Raspberry Piやカメラ配信ソフトから提供されるURLを quad.html 内で参照します。

例：

車庫映像：motion による配信URL
玄関映像：mjpg-streamer による配信URL

玄関映像の例：

http://192.168.0.16:8080/?action=stream
start_quad.sh

Chromium を全画面で起動し、quad.html を表示するためのスクリプトです。

テレビ表示用 Raspberry Pi で実行します。

bash start_quad.sh

このスクリプトを使うことで、ブラウザのメニューや余分な表示を出さず、家庭内監視盤のように画面を表示できます。

起動時に自動実行する設定を追加すれば、Raspberry Pi の電源投入後に自動で4分割画面を表示できます。

schedule.html

スケジュールを表示するためのHTMLファイルです。

quad.html の右下区画などに読み込ませることで、予定表や通知情報を画面の一部として表示できます。

家庭内の予定、来客予定、作業メモなどを表示する用途を想定しています。

update_schedule_api.py

Googleカレンダーなどから予定を取得し、schedule.html で表示するためのデータを更新するPythonスクリプトです。

予定情報を定期的に更新することで、テレビ画面上に最新のスケジュールを表示できます。

使用する場合は、GoogleカレンダーAPIの認証設定や、カレンダーIDなどを自分の環境に合わせて設定してください。

使用環境

このリポジトリは、次のような環境を想定しています。

・Raspberry Pi
・Raspberry Pi OS
・Chromium
・HDMI接続のテレビまたはモニター
・家庭内ネットワーク
・車庫カメラ映像
・玄関カメラ映像
・必要に応じてGoogleカレンダーAPI
基本的な使い方
1. Raspberry Pi にファイルをコピーする

このリポジトリのファイルを、テレビ表示用 Raspberry Pi にコピーします。

例：

git clone https://github.com/riverwest37/raspi-monitor.git
cd raspi-monitor

Gitを使わない場合は、GitHub画面の Code → Download ZIP からダウンロードして展開しても使用できます。

2. quad.html の表示内容を確認する

quad.html 内のカメラURLや表示先URLを、自分の環境に合わせて変更します。

例：

車庫映像URL
玄関映像URL
スケジュール表示URL
環境情報表示URL

IPアドレスが変わると表示できなくなるため、各Raspberry Pi のIPアドレスは固定しておくことをおすすめします。

3. Chromiumで4分割画面を起動する

次のコマンドで起動します。

bash start_quad.sh

正常に動作すると、Chromium が全画面で起動し、quad.html による4分割画面が表示されます。

4分割表示の考え方

quad.html では、HTMLとCSSを使って画面を4つの区画に分けています。

┌──────────────┬──────────────┐
│ 車庫映像      │ 玄関映像      │
├──────────────┼──────────────┤
│ 環境情報      │ スケジュール  │
└──────────────┴──────────────┘

この構成により、複数の情報を1つのテレビ画面にまとめて表示できます。

第4章・第6章・第7章との関係

このリポジトリは、KDP本の次の章と関係します。

第4章 カメラ監視システム構築
第6章 Node-REDによる可視化と制御
第7章 TV表示システム構築

第4章では、車庫映像と玄関映像を家庭内ネットワーク上で参照できるようにする仕組みを説明しています。

第6章では、MQTTイベントをNode-REDで受信し、テレビの電源ON、HDMI入力切替、一定時間後のOFF処理へつなげる流れを説明しています。

第7章では、本リポジトリのような4分割HTML画面を使い、テレビに家庭内情報をまとめて表示する仕組みを説明しています。

CEC制御について

テレビの電源ON、HDMI入力切替、一定時間後のOFF処理などのCEC制御は、第6章のNode-REDフロー側で扱います。

このリポジトリでは、主に テレビに表示する画面そのもの を扱います。

つまり役割分担は次の通りです。

Node-RED側：
MQTTイベント受信、CEC制御、テレビ操作

raspi-monitor側：
4分割HTML画面、映像表示、スケジュール表示
注意点
IPアドレスについて

カメラ映像やスケジュール表示のURLは、家庭内ネットワーク上のIPアドレスに依存します。

IPアドレスが変わると、quad.html から映像や情報を参照できなくなります。

そのため、車庫側Raspberry Pi、玄関側Raspberry Pi、表示用Raspberry Piは、ルーター側のDHCP固定割当などでIPアドレスを固定しておくことをおすすめします。

カメラ映像について

カメラ映像を扱う場合は、家庭内であってもプライバシーに配慮してください。

外部公開や遠隔監視を行う場合は、VPN、認証、ファイアウォールなどの安全対策が必要です。

GoogleカレンダーAPIについて

update_schedule_api.py を使用する場合は、GoogleカレンダーAPIの認証情報が必要になります。

認証情報やトークンファイルをGitHubに公開しないよう注意してください。

JBNetについて

本リポジトリは、家庭内IoT実験プロジェクト JBNet（爺さん婆さんNet） の一部として使用します。

JBNetでは、Raspberry Pi、Pico W、MQTT、Node-RED、カメラ、通知システム、テレビ表示などを組み合わせ、家庭内で使いやすいIoT環境を段階的に構築していきます。

ライセンス・利用について

このサンプルは、家庭内IoT学習および実験用です。

実際に使用する場合は、ネットワーク環境、カメラ映像の扱い、GoogleカレンダーAPIの認証情報、テレビ表示設定などを確認した上で利用してください。
